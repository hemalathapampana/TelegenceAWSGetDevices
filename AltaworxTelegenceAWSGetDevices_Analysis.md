## 5. Business Rules Documentation

### 5.1 Service Provider Processing Rules

1. **Sequential Processing**: Service providers are processed one at a time
2. **Fresh Start**: Staging tables are cleared for each new service provider
3. **Authentication Validation**: Processing stops if no authentication found

### 5.2 Device Processing Rules

1. **FAN Filtering**: Devices filtered by IncludedFANs/ExcludedFANs settings
2. **Status Validation**: Cancelled devices (status "C") are ignored
3. **Duplicate Prevention**: Devices are validated before insertion

### 5.3 Pagination Rules

1. **Page Limit**: `MaxCyclesToProcess` prevents infinite loops
2. **Time-based Cutoff**: Processing stops when remaining time < cutoff
3. **Batch Processing**: Fixed batch size of 250 (configurable)

### 5.4 Missing Device Rules

1. **Batch Processing**: Missing devices processed in groups
2. **Status Comparison**: Only devices with changed status are processed
3. **API Validation**: Each missing device validated via subscriber API
